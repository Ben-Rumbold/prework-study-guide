Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for a webpage.
  • The styles element defines the CSS styles associated with a webpage.
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the charater set, description, keywords, author and viewport.
  • The script element embeds client-side scriprts in the HTML document.
  • The p element represents a paragraph or block of text.
  • The h1, h2, h3, h4, h5 and h6 elements represent the level of heading a given text block represents.
  • The ul element defines an unordered list.
  • The ol element defines an ordered list.
  • The li element defines the list items.
  • The img element contains information about images that are displayed on the webpage.
  • The br element creates a line of empty text space, or line break, between two blocks of text.
  • The a (anchor) element containes links to the same or other webpages.
  • UTF-8 is a variable length character encoding standard.
  • Child elements are elements that are nested inside other elements such as the body element.
  • Section elements are semantic elemtns that define a section in a document that contains thematic content.
  • An element refers to the entire element from opening tag to closing tag.
  • A tag refers only to what appears inside the angle brackets.

CSS

  • CSS (Cascading Style Sheets) defines the colours, aesthetics, visual layout and organisation of a webpage.
  • Inline CSS is used when a developer wants to style an element directly in a HTML file.
  • An external CSS style sheet allows developers to keep all their CSS rules in a seperate file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A decleration contains two important components. 1: the CSS property we want to apply, and 2: the value of the property.
  • A selector defines the elements or attributes to which the rules, or declerations, apply.
  • A class attribute allows us to share a CSS rule to any element we choose by assigning the rule to a class attributewith a class selector.
  • 'Do Not Repeat Yourself' (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
